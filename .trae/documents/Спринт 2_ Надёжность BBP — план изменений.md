## Задачи
- Добавить таймаут и ретраи для вызова BBP бинаря в generatePairingsWithBBP.
- Валидировать конфигурацию BBP_PAIRINGS_BIN: явная причина при некорректном пути/отсутствии бинаря.
- Возврат причины ошибки: сохранить последнюю причину и добавить её в ответ API 502.
- (Опционально, под флагом) включить fallback на простой Swiss в dev-режиме.

## Реализация
- В lib/bbp.ts: внедрить таймаут (env BBP_TIMEOUT_MS), ретраи (env BBP_RETRIES), валидатор resolveBbpBinary и хранение последней причины (getLastBbpReason).
- В app/api/.../pairings/route.tsx: добавить reason в JSON при 502, сохраняя текущий текст ошибки.
- Включить fallback simpleSwissPairings при BBP_DEV_SWISS_FALLBACK=1 и dev-режиме.

## Критерии
- 502 только при реальном падении/пустом результате после ретраев.
- Ретраи не нарушают идемпотентность: повторные запуски не создают дубликаты.
- В dev при флаге — успешная генерация через простой Swiss.
